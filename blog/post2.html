Cómo convertir un Arduino Uno R4 Minima en un Rubber Ducky (emulador de teclado HID)

Publicado el 18 de junio de 2025
Por Iago Lorenzo Manrique

En esta entrada os explico paso a paso cómo transformé un Arduino Uno R4 Minima en un Rubber Ducky, es decir, un dispositivo capaz de emular un teclado y ejecutar comandos automáticamente al conectarse a un ordenador.

⚠️ Este proyecto es solo para fines educativos. No está permitido usarlo en sistemas sin autorización explícita.

¿Qué es un Rubber Ducky?

Un Rubber Ducky es un dispositivo USB que se comporta como un teclado. Al conectarlo a un sistema, puede enviar pulsaciones de teclas como si las escribiera un humano, pero mucho más rápido y automáticamente.

⚙️ ¿Por qué el Uno R4 Minima?

Aunque el Uno R4 Minima no tiene soporte HID nativo, podemos usar un firmware y la librería Keyboard.h para simular un teclado y enviar comandos.

Materiales

  • Arduino Uno R4 Minima
  • Cable USB-C
  • Arduino IDE
  • Teclado del sistema en inglés (US)
  • Librería Keyboard.h instalada

Paso 1: Preparar el entorno

Instala el soporte de la placa Uno R4 en el Arduino IDE.

Paso 2: Instalar la librería Keyboard

Desde el IDE de Arduino: Sketch > Incluir Librería > Gestionar bibliotecas y busca Keyboard.

Paso 3: Sketch de ejemplo

Este script abrirá PowerShell y ejecutará ipconfig:

#include <Keyboard.h>

void setup() {
  delay(3000); // Espera a que el sistema detecte el HID

  Keyboard.begin();

  // Abrir "Ejecutar"
  Keyboard.press(KEY_LEFT_GUI);
  Keyboard.press('r');
  delay(100);
  Keyboard.releaseAll();
  delay(300);

  // Escribir "powershell"
  Keyboard.print("powershell");
  Keyboard.press(KEY_RETURN);
  Keyboard.releaseAll();
  delay(800);

  // Ejecutar comando
  Keyboard.print("ipconfig");
  Keyboard.press(KEY_RETURN);
  Keyboard.releaseAll();
}

void loop() {
  // Nada aquí
}
⚠️ Asegúrate de que el sistema objetivo tenga el layout de teclado en inglés (US), o los caracteres pueden no coincidir.

Consejos

  • Usa delay() para evitar errores por velocidad excesiva.
  • Puedes modificar el script para hacer tareas más complejas: lanzar scripts, abrir navegadores, conectar shells, etc.
  • Evita entornos protegidos con UAC o antivirus que bloqueen el dispositivo.

Conclusiones

El Uno R4 Minima, aunque no fue pensado para HID, puede convertirse en un Rubber Ducky funcional para pruebas de seguridad y CTFs. Una opción barata, flexible y potente para quienes entrenamos ciberseguridad.
